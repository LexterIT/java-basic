collection - object na nag store at manipulate ng group of objects.

map is collection na nag sstore ng key value pair, unordered elements bawal duplicate keys pede sa values pag may na detect na duplicate papalitan ng pinaka bago ung duplicate nya na luma.

hashmap is basic implementation ng map nag sstore ng key value pair, unordered elements, 1 null key many null values.

linkedhashmap - halos kapareho ng hashmap, except ung elements is ordered based sa insertion order ng elements.

treemap - naman same thing as hashmap din, except sa treemap pedeng mag lagay ng specific order ng elements or may natural order din which is ascending order ng elements.

sets - naman is nag sstore ng unique elements, meaning walang duplicate element. unordered elements din. pag may na detect na duplicate, hindi na iinsert yung may duplicate.

Hashset bali basic implementation din ng set, unique elements, unordered. 

linkedhashset same lang din ng hashset except ordered elements based sa insertion order

treeset is samething except pede mag specify ng order ng elements 

List is collection na nag sstore naman ng mga elements na pede duplicates, tas ung order ng elements nya is based sa insertion order. na aaccessed gamit yung index. 

Arraylist simple implementation ng list duplicate elements allowed, insertion order. mas mabilis sa get and set (random access)

LinkedList - halos same ng arraylist, except each nodes sa linkedlist nag mmaintain ng reference sa previous and next node. 

Inheritance eto po yung concept na nag aallow ng subclass to get everything from superclass. except sa mga privates. Dito ginagamit yung extends at implements.

extends ginagamit po ito para ma inherit ng subclass ung superclass. Ginagamit din siya ng interface para maka inherit ng superinterfaces.

Implements ginagamit naman po to ng class para makapag inherit ng interfaces.

Encapsulation eto naman po yung pag tatago ng properties ng isang class, bali ung mga method lang po ng class na yun ung makakagalaw sa properites. (get/set) methods accessor.

Abstraction - eto naman po yung pag gamit ng abstract keyword. Parang nag bibigay ng idea sa subclass kung ano yung mga method 
na dapat i-implement

abstract keyword pag ginamit po siya sa method, makakagawa ng abstract method tas yung abstract method po is method na walang body, bali yung subclass ung mag ssuply ng implementation. Pag ginamit naman sa class magkakaruon ng abstract class. Abstract class yung mga class na merong abstract method and concrete methods.

Interface vs abstract - ung interface naman parang pareho lang din ng abstract except sa abstract po pedeng may concrete methods meaning hindi needi override lahat, other than abstract methods. sa Interface naman po lahat abstract method meaning yung subclasses po need ioverride lahat ng method nya.

Polymorphism - nag aallow po satin na mag bigay ng ibat ibang form sa isang reference type/declaration or sa isang method gamit ung overloading and overriding

Overloading eto po yung pag sa iisang class lang, pareho ng name pero iba iba yung parameters ung dami or ung data types or pag kasunod sunod.

Overriding eto naman po yung pag from superclass, same name at same parameters.

throws eto po yung nilalagay sa signature ng method, ang ginagawa po ng throws is mag wwarn sya sa programmer about sa mga exceptions na pedeng mangyare sa method, at dapat i catch ng program.

throw eto naman po yung nasa loob ng method, bali mag ccause po siya ng exception na inspecify ng programmer.
